# QQ_agentshow 发布需求与验收标准（唯一事实真相）

> 状态：维护中
> 适用范围：仓库源码、`QQ_agentshow` Skill、macOS 安装包、GitHub 主页与 Release
> 规则：任何实现、文档、测试或发布结论与本文件冲突时，以本文件为准；每次发布前必须逐项更新证据。

## 1. 产品目标

让一个从未安装过 QQ_agentshow 的 macOS 用户，可以直接把公开仓库交给 Codex，依照根目录 `SKILL.md` 完成审阅、安装、配置、验证、更新与恢复，并获得可真实工作的 QQ2006/2007 风格 Agent 工作台。

成功不是“维护者本机看起来可用”，而是以下完整用户路径均有直接证据：

1. 用户能在 GitHub 主页理解产品、风险、支持范围和安装方式。
2. Codex 能从 `SKILL.md` 找到所有必要步骤，不依赖私有上下文、维护者路径或隐藏服务。
3. 安装前检查能发现不支持的平台、缺失的官方应用、未初始化的 Codex 配置和未退出的 Codex。
4. 首次安装、重复安装和版本更新不会丢失个人配置，不会修改官方应用包、签名、会话、API Key、Base URL 或模型设置。
5. 安装后能验证主题、真实数据右栏、多图、声音、企鹅动作与原生 Codex 关键控件。
6. 用户可以暂停、切回原生界面并完整恢复；失败时有可执行的排错路径。
7. 公开仓库、截图、日志、发行包不包含个人信息、凭据或未说明来源的素材。

## 2. 支持边界

- 平台：macOS，Apple Silicon 与 Intel。
- 应用：官方 ChatGPT / Codex Desktop；不承诺支持第三方重打包版本。
- 运行时：只使用官方应用内签名且经过校验的 Node.js。
- 网络：安装时从公开 GitHub 下载；运行时不需要远程服务。
- 注入：CDP 仅绑定数值回环地址，必须校验端口归属、官方签名和 renderer 目标。回环阻止局域网访问，但端口没有独立认证，同一 macOS 用户权限下的本地进程仍可能访问 renderer。
- 禁止：修改 `ChatGPT.app`、`Codex.app`、`app.asar`、代码签名、会话、API Key、Base URL、供应商或模型配置。

## 3. 陌生用户的 Skill 安装契约

根目录 `SKILL.md` 必须明确且可执行地覆盖：

1. 先阅读安装要求并运行 `./install.sh --check`。
2. 首次安装前正常启动并退出一次官方 Codex，并要求用户完全退出 Codex。
3. 默认安装使用固定正式版本，而不是不可复现的移动分支内容。
4. 安装前说明将写入的目录、配置备份、桌面入口和本地调试端口。
5. 安装后运行验证命令；不能只以脚本退出码或“界面变蓝”作为成功证据。
6. 配置图片时校验格式、尺寸、透明度与路径；拒绝把带方形背景的宠物伪装成抠图。
7. 更新必须保留用户的活动主题、昵称、签名、布局、隐私模式、自定义企鹅和 QQ 秀。
8. 失败、更新不兼容或用户反悔时，提供无需手工改应用包的恢复命令。
9. 面向公开截图或直播时，主动使用 `masked` 或 `off`，并先跑隐私检查。

## 4. 产品行为要求

### 4.1 原生能力不得破坏

必须保留 Codex 原生项目、任务、消息、Markdown、代码块、Diff、审批、附件、模型选择、权限选择、语音与发送控件。主题只能添加样式、语义标记和可移除的辅助 UI。

### 4.2 右栏与真实数据

- 顶部为 `Codex 好友` / `环境信息`，右上标题固定为“Codex 江湖传说”，不是用户名。
- 默认布局依次包含企鹅资料、真实最近对话、环境信息、最近产物、QQ 秀；不得伪造好友、回复、任务数、产物或来源。
- `classic-chat`、`workbench`、`minimal` 三套模板必须可配置；右栏能展开、收起和关闭，不能留下大面积无意义空白。
- 用户名从本机 Codex 资料读取；QQ 签名使用可配置模板；等级使用皇冠、太阳、月亮、星星表达而不是 `LVxx` 标签。
- QQ 秀公开界面只提供经典女、经典男两个模板，不显示“我的”或维护者个人入口；仍允许通过本地配置使用用户自有图片。

### 4.3 新任务与多图

- 新任务首页保持“输入框 → 项目与 Plugins → 快捷建议”的原生顺序，整体上移但不得重叠、越界或造成底部断层。
- 左栏 `Recents` 必须显示真实最近任务或清晰零状态，不能只有标题和大片空白。
- 多图发送前每张图独立显示、保留删除按钮并可滚动；发送后保持比例进入响应式画廊。
- 主题不得复制、替换、删除或重新编码原生附件数据。

### 4.4 企鹅与声音

- 内置企鹅必须是带透明像素且可见的 PNG；不能显示方形底图。
- `calm`、`playful`、`off` 三种动作模式有效，并尊重系统“减少动态效果”。
- 企鹅可随机呼吸、点头、挥手、探头、踱步、跳跃；运行中有工作动作，完成时有庆祝反馈。
- 主 Agent 从运行转为完成时播放完成声；出现原生人工确认操作时播放确认声。
- 同一事件只提醒一次；后台、隐藏或未聚焦的窗口不播放；两种声音可试听和关闭。
- 发布 WAV 必须为短音频、PCM 16-bit、单声道 44.1 kHz，无硬削波，首尾接近零，保留播放余量且没有明显电流声或爆音。

## 5. 隐私、安全与开源要求

- 公开树和发行包不得包含维护者用户名、用户目录、邮箱、令牌、私钥、真实对话、真实项目名或未脱敏截图。
- `real` 模式只读当前本地 DOM；不得上传、持久化或写入仓库。`masked` 与 `off` 行为必须明确。
- 所有外部来源、AI 生成素材、维护者提供素材和再加工音频必须记录来源、处理方式与权利边界。
- 软件许可证、上游归属和非官方声明必须在 README、LICENSE、NOTICE 与发行包中一致。
- GitHub 必须为公开仓库，主页描述、Topics、README、封面、脱敏实机图、最新版徽章和安装命令必须与当前 Release 一致。

## 6. 安装、更新、恢复与打包门槛

- `install.sh --check`：只读预检，不得安装或修改配置。
- 一键安装：固定到当前正式 tag；下载失败、tag 不存在或文件缺失时必须明确失败。
- 安装器：先部署到临时目录再原子替换；不得把 `.git`、本地 runtime、release ZIP 或私有状态带入安装目录。
- 更新：不得覆盖活动主题与用户自定义素材；若迁移不可避免，必须备份并有测试。
- 状态：热更新复用现有监听器时必须刷新 PID、版本和启动时间，避免误报 `stale`。
- 恢复：只终止身份完全匹配的注入器；恢复基础主题后不自动删除用户素材；提供可验证结果。
- 发行包：可完整解压，包含 Skill、NOTICE、声音、预设、验证与恢复脚本；不包含本机路径或秘密；提供 SHA-256。

## 7. 必须通过的验证

每个正式版本至少需要以下直接证据：

1. `cd macos && npm test`
2. `npm run privacy`
3. `npm run skill:check`
4. `./scripts/doctor-macos.sh`，且官方签名有效、`modifiesAppAsar=false`、实时 CDP 验证通过。
5. 真实 Codex 的 deep/native 生命周期烟测。
6. 新任务页在 100%、125%、150% 与矮窗口下的布局矩阵。
7. 多图 fixture 覆盖上传前、删除和发送后画廊；如 renderer/CSS 变化，还需真实任务页验收。
8. 两段音频的格式、削波、首尾和瞬态门槛测试；至少一次使用发布内嵌音频的真实 Web Audio 试听。
9. 发行 ZIP 完整性、版本、声音存在性、隐私和 SHA-256 校验。
10. 从公开 `main/install.sh` + 当前正式 tag 运行陌生用户 `--check`。
11. GitHub CI 通过，Release 非草稿、非预发布，tag 指向已审查的发布提交；发布后的纯文档更新可以继续进入 `main`，但安装器仍必须固定到已发布 tag 与校验和。

## 8. 三 Agent 独立审查门槛

发布结束前必须由三个相互独立的 GPT-5.6 ultra 审查者检查同一提交：

1. **安装与 Skill 审查**：以首次安装、更新、失败恢复和无维护者上下文为重点。
2. **安全与开源审查**：以供应链、进程边界、隐私、许可证、素材与 GitHub 公共面为重点。
3. **产品与回归审查**：以 UI、真实数据、多图、声音、企鹅、响应式和原生能力保留为重点。

每位审查者必须基于文件、命令或运行结果给出结论。存在任何 P0/P1/P2 发布阻断项即为未通过；修复后必须让同一审查者复验。只有三者对最终提交都明确给出 `PASS`，且主 Agent 完成本文件逐项审计，才允许发布和结束任务。

## 10. 发布完成定义

只有同时满足以下条件，才可宣布完成：

- 本文件第 1–7 节的每一项均有与范围匹配的当前证据。
- 三位审查者对最终提交均明确 `PASS`，没有遗留发布阻断项。
- 所有修复已进入公开 `main`，CI 通过。
- GitHub 主页和最新正式 Release 已更新，公开安装预检通过。
- 发布时本地工作树干净，正式 tag 指向已审查发布提交；纯文档后续提交不得改变安装器固定 tag 或 Release 校验和。
